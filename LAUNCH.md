# Launch Concept

## Goal

Launch a useful artifact, not a hype thread.

The launch should make `websitemd.org` feel like the home of an open convention: simple, useful, inspectable, and ready for agents.

## First Move

1. Buy `websitemd.org`.
2. Publish the repo.
3. Publish `create-website-md` to npm.
4. Smoke test `npm create website-md@latest my-site`.
5. Import the repo into Vercel.
6. Attach `websitemd.org`.
7. Build `websitemd.org` with its own `WEBSITE.md`.
8. Record one 60-second demo.
9. Explain the mental model clearly.

## Launch Assets

1. GitHub repo.
2. `websitemd.org` landing page hosted on Vercel.
3. Draft spec.
4. Published `create-website-md` starter generator.
5. One 60-second demo video.
6. One case study: replacing a hosted website builder with an AI-operated site.

## Homepage Outline

Headline:

```txt
The website editor is disappearing.
```

Subhead:

```txt
WEBSITE.md tells your AI agent how to build, edit, preview, validate, and publish your website.
```

Quick start:

```bash
npm create website-md@latest my-site
```

Main sections:

- What changed.
- How it works.
- File model.
- Quick start.
- 60-second demo.
- Examples.
- Spec.
- GitHub.
- Open `.org` note.

## `.org` Framing

Use:

```txt
Website.md is an open convention for AI-operable websites.
```

Avoid:

```txt
The easiest AI website builder.
```

The standard should be free. Future commercial surfaces can exist around it: validation, reports, templates, migration help, certification, managed publishing, courses, and consulting.

## Demo Script

1. Create starter.
2. Open in Codex or another agent.
3. Prompt agent to build a website.
4. Agent edits files.
5. Agent previews locally.
6. Agent fixes layout.
7. Agent runs checks.
8. Show deploy-ready result.

## Private Validation

Before public launch, send to:

- 5 AI-agent builders,
- 5 indie founders,
- 5 small-agency developers,
- 5 non-technical site owners who use AI.

Ask:

- Do you understand this in one minute?
- Would you use it?
- What would you expect the command to do?
- What feels missing?
- Is `WEBSITE.md` the right name?
