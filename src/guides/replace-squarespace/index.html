Website.md is useful when the site is simple enough to become source files, Markdown content, local previews, and a deploy pipeline.

  1. Inventory current URLs.
  2. Draft WEBSITE.md, DESIGN.md, COPY.md, and SEO.md.
  3. Rebuild the site static-first.
  4. Preserve URLs or create redirects.
  5. Run build, link, metadata, and mobile checks.

Do not migrate forms, analytics, domains, or payments without explicit approval.
