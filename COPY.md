# COPY.md

## Voice

Plain, confident, and founder-practical.

The copy should sound like someone who discovered a new workflow by doing real work, not like a SaaS marketing team inventing a category.

## Core Message

The website editor is disappearing.

`WEBSITE.md` gives AI agents the context they need to build, edit, preview, validate, and publish websites.

This should feel open, boring, trustworthy, and obvious. The project is naming a layer, not selling a dashboard.

## Audience

Primary:

- developers and founders using AI coding agents,
- agencies building simple client sites,
- people tired of hosted website-builder rent.

Secondary:

- non-technical site owners who may use an agent through a helper or developer.

## Copy Rules

Use:

- short sentences,
- concrete claims,
- working commands,
- before/after language,
- practical examples,
- bold first-principles framing when it is backed by a simple demo.

Avoid:

- hype without proof,
- "revolutionary",
- "10x" unless attached to a concrete workflow,
- vague "AI-powered" phrasing,
- enterprise jargon,
- pretending this replaces every CMS,
- pricing, plans, or SaaS packaging unless explicitly requested.

## Approved Positioning Lines

- The website editor is disappearing.
- Website instructions for AI agents.
- All your AI agent needs to operate a website.
- Your website is a repo. Your AI is the editor.
- Stop renting a website editor.
- Give your AI a website runbook.

## Approved Comparisons

Use the WordPress comparison carefully:

```txt
WordPress.org is the open project.
WordPress.com is the hosted product.
Website.md should start as the open convention.
```

Do not imply Website.md already has WordPress-scale adoption.

## Core Explanation

Use this paragraph when space allows:

> `WEBSITE.md` is a Markdown instruction file for AI coding agents. It explains what your website is, where content lives, which URLs matter, how to preview changes, what to check, and how to publish safely. Pair it with `DESIGN.md`, `COPY.md`, and `SEO.md`, then let your AI agent operate the site.

Short version:

> All you need to launch a simple website is a folder, `WEBSITE.md`, and an AI agent that can edit, preview, validate, and publish.



